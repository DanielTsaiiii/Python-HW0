# Python-HW0
寫下你最喜歡的其中幾句 Python 禪學並簡短說明:

(1.)If the implementation is hard to explain, it's a bad idea.If the implementation is easy to explain, it may be a good idea.
說明(1.):這句話我覺得很有趣也很重要，在編寫程式個過程中，如果寫得難以去解釋為何如此寫，其他人在理解時需要花更多時間，(想法或許也會被受影響)，會影響到工作時或思考時的流暢度，
而且如果能用更簡單扼要的方式來編寫，看起來也比較舒適。(能被簡易說明的程式，才是好程式)

(2.)Readability counts.
說明(2.):在學程式的過程中，有些語法還看不懂，或是還沒有釐清「為何這樣寫」，如果用簡易的方式，好理解的思維來編寫，會吸收得比較快，或是上頭加上註解，每次看到旁邊有##時都會松一口氣
，因為一個步驟配上一個註解，更能跟得上作者的思路。

(3.)Special cases aren’t special enough to break the rules.Although practicality beats purity.
說明(3.):就算實用性比純粹性更重要，但特例還沒有特別到能違反規則，我認為程式語言就是在跟電腦溝通，像跟人溝通一樣，如果今天跟人家溝通都是用自創的文法，自創的發音，沒有人聽得懂，規則還是必須存在。
